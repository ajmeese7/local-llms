06Methodology & Caveats

// how the numbers were produced — and what they don't mean

Timings

All five profiles were tested through llama-server's OpenAI-compatible /chat/completions endpoint with stream: false. Reported latency is end-to-end response time including prompt processing, generation, and server overhead.

TTFT caveat. The raw report column labelled ttft_sec is curl's time_starttransfer, but because requests were non-streaming this is effectively full-response time. Treat it as latency, not first-token. This page omits the misleading column.

Quality rubric

Quality is a lightweight automated keyword/requirement rubric scored per prompt, with a max score that varies by prompt (e.g. 5/5 on bugfix, 6/6 on shell, 4/4 on creative). Quality % is the average ratio across the five prompts.

  • Not a human eval, taste judgment, or correctness proof
  • Rewards format compliance & required keywords
  • Penalises truncation and missing requirements

Reasoning leakage

Several profiles emit visible <think> blocks or "Here's a thinking process:" preambles in their content field. Some emit empty content and place the answer in reasoning_content — the rubric falls back to that field but it changes how the response renders in a chat UI.

This affects real usefulness and is surfaced as a per-prompt indicator on the cleanliness strip, the profile table, and the drilldown excerpts.

Raw artifacts

Every response is preserved on disk for audit. Full paths are linked in the prompt drilldown.

  • report.md — agent-friendly text report
  • results.jsonl — one record per prompt × profile
  • manifest.tsv — profile lifecycle
  • raw/<profile>/<prompt>.response.json — raw API JSON
  • raw/<profile>/<prompt>.result.json — derived metrics
  • prompts/prompts.json — exact input prompts

Profile differences that matter

  • q4-ngram trades quant precision (Q4_K_P vs Q5_K_P) for speed and uses parallel=4 + f16/f16 KV + the experimental ngram modifier
  • 27B profiles run at 262k context but only one parallel slot — long-context recall is correct but slow
  • mythos uses the PRISM-DQ GGUF and produces a distinctly different writing voice — useful for creative work, less optimal for coding

What this report is not

  • Not a head-to-head against hosted frontier models
  • Not a substitute for a project-specific eval
  • Not a guarantee these numbers reproduce on different driver / kernel versions
  • Not corrected for prompt-token cost variance (coding prompts are short; long-context is ~5k tokens)
